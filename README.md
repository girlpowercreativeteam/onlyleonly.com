# Шаблон редиректов для GitHub Pages

Этот репозиторий — для копирования. Для настройки нового редиректа нужно загрузить эти файлы в новый репозиторий и изменить содержимое по инструкции ниже

## Что нужно изменить

### 1. Домен

Откройте файл `CNAME` и замените `domain.com` на ваш настоящий домен.

Пример:

```text
example.com
```

Этот же домен нужно использовать во всех HTML-файлах:

```html
https://example.com/
https://example.com/photo.webp
https://example.com/ig/
```

### 2. Главная страница

Откройте файл `index.html`. Он используется, когда человек открывает основной домен:

```text
https://domain.com/
```

Замените в файле эти значения:

- текст в `title`
- текст в `description`
- `og:title`
- `og:description`
- `og:url`
- `og:image`
- ссылку для редиректа в `meta http-equiv="refresh"`
- ссылку для редиректа в `var targetUrl`
- ссылку для редиректа внутри блока `<noscript>`

Все три ссылки для редиректа в одном файле должны быть одинаковыми.

### 3. Картинка для превью

Замените файл `photo.webp` на вашу картинку-баннер.

Рекомендации:

- Используйте изображение в формате `.webp`, `.jpg` или `.png`.
- Держите картинку в корне репозитория.
- Если вы переименовали картинку, обновите `og:image` во всех HTML-файлах.
- В `og:image` указывайте полный URL картинки, например:

```html
<meta property="og:image" content="https://domain.com/photo.webp" />
```

### 4. Короткие ссылки

Каждая короткая ссылка — это отдельная папка со своим файлом `index.html`.

В шаблоне уже есть примеры:

```text
ig/index.html  -> https://domain.com/ig/
tg/index.html  -> https://domain.com/tg/
```

Чтобы добавить новую короткую ссылку:

1. Скопируйте папку `ig`.
2. Переименуйте скопированную папку в нужный путь.

Примеры:

```text
x/index.html        -> https://domain.com/x/
onlyfans/index.html -> https://domain.com/onlyfans/
```

3. Откройте файл `index.html` внутри новой папки.
4. Замените заголовок, описание, `og:url` и ссылку для редиректа.

Важно: название папки становится публичным параметром в ссылке. Если папка называется `ig`, публичная ссылка будет такой:

```text
https://domain.com/ig/
```

### 5. Неизвестные ссылки

Файл `404.html` нужен только как запасной вариант для неправильных или отсутствующих ссылок.

Пример:

```text
https://domain.com/wrong-link
```

Откройте `404.html` и укажите запасную ссылку для редиректа. Обычно это та же ссылка, что и редирект с главной страницы.

# Чеклист перед публикацией

- GitHub Pages включен для репозитория.
- В `CNAME` указан настоящий домен.
- DNS-записи домена настроены на GitHub Pages.
- В `index.html` указана правильная ссылка для главного редиректа.
- Для каждой короткой ссылки есть отдельная папка со своим `index.html`.
- Во всех HTML-файлах указан правильный домен в `og:url`, `canonical` и `og:image`.
- Файл `photo.webp` существует, и его имя совпадает с именем в `og:image`.
